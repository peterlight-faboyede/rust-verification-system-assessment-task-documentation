Security

Zero trust at the edge, payload defense, database posture, containers, and leak prevention.

Edge security

API key authentication uses constant-time comparison, not standard string equality. Standard comparison fails the moment it hits a wrong character - measurable response time differences let attackers guess keys byte by byte. Constant-time comparison takes identical time regardless of where the mismatch occurs, eliminating the attack vector entirely.

Idempotency keys backed by Redis mean duplicate session-start requests - whether from a glitchy network or a deliberate flood - return the cached response. The database sees exactly one write regardless of how many requests arrive.

Rate limits and CORS origins are environment variables. Under an active attack, both can be tightened without touching or redeploying code.

Payload defense

ValidatedJson enforces a two-phase gate before any handler runs. Serde validates structure first - wrong types are rejected. The validator crate then enforces field rules: string length caps prevent memory exhaustion, strict enums reject anything outside the allowed set. Phone numbers go through libphonenumber - regex is not sufficient for international formats.

If a crafted payload crashes the worker, the lease recovery system tracks it. After three crashes, the payload is permanently quarantined as FAILED and removed from active processing. The cluster is protected from infinite crash loops.

Database security

UUID primary keys carry 122 bits of entropy - sequential IDs make enumeration trivial, UUIDs make it computationally impossible. No user_id is stored anywhere in the verification tables. If the database is breached, the attacker gets fragmented verification data with no way to link it to real user accounts. The session UUID is the only credential.

All SQL uses parameterized queries verified at compile time against the actual schema. SQL injection is not possible - there is no dynamic query construction.

input_data is written once by the API, response_data written once by the worker. Neither can be modified after the fact. The audit trail is structurally immutable.

Container hardening

Production images are built on distroless base - no shell, no package manager, no system utilities. Remote code execution via a zero-day lands an attacker in a container with nothing to work with. The Rust binary runs as PID 1 directly.

Database migrations run in a separate short-lived Alpine container that exits immediately after applying the schema. The API and worker pods have no schema-altering privileges and no migration code.

Leak prevention

All secrets - database URL, Redis URL, AMQP URL, API key - are wrapped in secrecy::SecretString. Accidentally printing the config struct outputs [REDACTED]. Secrets cannot reach logs regardless of developer error.

Internal 500 errors log the full technical detail internally. The client receives only {"error": "internal_error", "message": "An unexpected error occurred", "trace_id": "..."}. Database schema names, table structures, and stack traces never leave the server.
