Environment variables

What they control and why they're tunable.

Runtime

  • ENVIRONMENT - Switches log format. local gives readable pretty logs; anything else outputs compact JSON for log pipelines. Don't run local in production - it's verbose and expensive to ingest.
  • SERVER_PORT - TCP port the API binds to. Exists so infrastructure can remap ports without touching code.
  • RATE_LIMIT_REQUESTS_PER_MIN - Per-IP request cap enforced in memory. Increase during expected traffic spikes to reduce false 429s. Decrease during an active attack to throttle an abusive IP faster.
  • CORS_ALLOWED_ORIGINS - Empty disables CORS entirely. Populated values are parsed at boot - a typo crashes the app immediately rather than silently breaking browser traffic in production.

PostgreSQL - Main Pool

  • DB_POOL_MIN / MAX - Min keeps warm connections ready; max caps total connections from this process. With 50 API pods at max 20, you're using 1,000 connections - that hits Postgres's global limit fast. Increase max for more parallelism; Decrease if running many replicas against one DB.
  • DB_POOL_ACQUIRE_MS - How long a request waits for a free connection before returning a 500. Too high and requests pile up in RAM until the pod OOMs. Too low and you get premature 500s under short spikes.
  • DB_POOL_IDLE_MS - How long idle connections are kept before being closed. Increase to reduce reconnect churn; Decrease to release Postgres sessions faster on quiet nodes.

Outbox relay

  • RELAY_DB_POOL_MIN / MAX - Completely isolated from the main pool. A hung RabbitMQ connection can only exhaust these 5 - the API and processors keep running. Increase if the relay needs more parallel throughput; Decrease to minimize relay's footprint.
  • OUTBOX_RELAY_BATCH_LIMIT - Rows claimed per relay transaction. Increase to clear backlogs faster with fewer round-trips. Decrease to shorten how long row locks are held per cycle.
  • OUTBOX_RELAY_PUBLISH_TIMEOUT_MS - Hard timeout per RabbitMQ publish. The relay holds Postgres row locks while publishing - a hung broker would freeze those locks indefinitely without this. Increase to tolerate slower brokers; Decrease to release locks faster on bad networks.
  • OUTBOX_RELAY_IDLE_POLL_MS - Sleep duration when the outbox is empty. Increase to stop idle polling from burning DB CPU. Decrease if you need lower latency picking up new events.

Stage processor

  • PROCESSING_LEASE_BUFFER_SECS - Added to the calculated work time to set expires_at. Increase to prevent the recovery worker from stealing jobs that are still legitimately running. Decrease and leases expire prematurely, causing unnecessary recoveries.
  • STAGE_SIMULATED_WORK_SECS_MIN / MAX - Local testing knobs only. Set both to 1-5 seconds for fast end-to-end runs. Leave high to stress-test lease and retry behavior.
  • STAGE_PROCESSING_MAX_RETRIES - Controls three things: RabbitMQ retry routing, DB retry count, and lease recovery quarantine. Increase to give more attempts before permanent failure. Decrease to quarantine bad jobs sooner.
  • STAGE_PROCESSOR_QOS_PREFETCH - How many unacked messages RabbitMQ pushes to the worker at once. Increase for higher throughput. Decrease to reduce memory pressure and DB concurrency per pod.
  • STAGE_PROCESSOR_SHUTDOWN_GRACE_MS - Time given to in-flight jobs on SIGTERM. Increase if jobs regularly need more time to finish writes. Decrease for faster pod shutdown at the cost of more mid-job nacks.
  • WORKER_DB_TIMEOUT_MS - Max time for a DB operation inside the worker. Increase if legitimate transactions are being cut off. Decrease to surface hung SQL faster.

Lease recovery

  • STAGE_LEASE_RECOVERY_POLL_MS - How often the recovery worker scans for expired leases. Increase to reduce DB load when the system is healthy. Decrease to detect crashed jobs faster.
  • STAGE_LEASE_RECOVERY_BATCH_LIMIT - Recovery rows per transaction. Increase to clear a mass failure faster with fewer transactions. Decrease to keep lock contention low under recovery.
  • WORKER_START_JITTER_MAX_MS - Random delay before first poll on boot. Increase to spread load across more pods. Set to 0 to remove jitter entirely for predictable cold starts.

Infrastructure

  • AMQP_URL / REDIS_URL / API_KEY - No defaults. Missing values crash the app at boot - intentionally. Silent misconfiguration in production is worse than a loud startup failure.
  • IDEMPOTENCY_TTL_SECS - How long a session-start idempotency key lives in Redis. Increase to extend the safe client retry window. Decrease to expire keys sooner and reduce Redis memory usage.
  • RUST_LOG - Log verbosity dial. info keeps costs down. Inject debug or trace per crate for deep troubleshooting without redeployment.
